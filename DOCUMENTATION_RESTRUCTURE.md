# Реструктуризация документации

## Выполненные изменения

### 1. Разбиение большой страницы издателя

Старая структура:
```
/crm/publisher.mdx (одна большая страница ~600 строк)
```

Новая структура:
```
/crm/publisher/
  ├── index.mdx          - Обзор и навигация
  ├── sites.mdx          - Управление сайтами
  ├── statistics.mdx     - Статистика и аналитика
  ├── finances.mdx       - Баланс и выплаты
  ├── integration.mdx    - Интеграция виджета
  ├── settings.mdx       - Настройки профиля
  └── faq.mdx           - Частые вопросы
```

### 2. Удаление выдуманного контента

На основе анализа реального кода CRM:

**Удалено из документации:**
- Все функции рекламодателя (создание кампаний, таргетинг, бюджет)
- Все функции агентства (управление клиентами, комиссии)
- Функции администратора
- Тестовые аккаунты
- Двухфакторная аутентификация (пока не реализована)
- API ключи (пока не реализованы)
- Автопополнение баланса

**Оставлено (реально реализовано):**
- Регистрация и вход через Zitadel OAuth2
- Управление сайтами издателя
- Добавление сайта (3 шага)
- Верификация домена через TXT запись
- Получение Site Key и Secret Key
- Статистика (показы, клики, CTR, доход, RPM)
- Просмотр баланса
- Запрос на вывод средств (минимум 500 ₽)
- Пополнение баланса (минимум 100 ₽)
- Интеграция виджета
- Настройки профиля и организации

## Преимущества новой структуры

1. **Удобная навигация** - легко найти нужную информацию
2. **Быстрая загрузка** - страницы меньше по размеру
3. **Легкое обновление** - можно редактировать отдельные разделы
4. **SEO оптимизация** - каждая страница с уникальным заголовком
5. **Честность** - документация соответствует реальному функционалу
