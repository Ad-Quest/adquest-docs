### Options

While initializing the SDK, you will usually only need to pass the `authToken` option.

```js
{
    authToken: '<participant_auth_token>',
}
```

But you can also pass other options to control the media quality and other optional settings.

```js
{
    authToken: '<participant_auth_token>',
    defaults: {
        audio: false, // Defines if audio should be enabled by default for the participant.
        video: false, // Defines if video should be enabled by default for the participant.
        screenShare: { // Defines the default screen share constraints for the participant.
            displaySurface: 'monitor',
        },
        mediaConfiguration: {
            video: { // Defines the default video constraints for the participant.
                width: { ideal: 1920 },
                height: { ideal: 1080 },
                frameRate: { ideal: 15 },
            }
        }
    },
    modules: {
        devTools: {
            logs: true, // If true, the SDK will log all the internal logs to the console.
        },
    },
    onError: (error) => {
        console.error('RealtimeKit SDK Error::', error);
    },
    overrides: {
        simulcastConfig: {
            disable: false, // If true, simulcast will be disabled.
            encodings: [ // Defines the simulcast encodings for the participant.
                {
                    rid: "f", // full / highest quality
                    scaleResolutionDownBy: 1.0,
                    maxBitrate: 2500000, // ~2.5 Mbps
                },
                {
                    rid: "h", // half
                    scaleResolutionDownBy: 2.0,
                    maxBitrate: 900000, // ~0.9 Mbps
                },
                {
                    rid: "q", // quarter
                    scaleResolutionDownBy: 4.0,
                    maxBitrate: 250000, // ~0.25 Mbps
                }
            ]
        },
        forceRelay: false, // If true, the participant will be forced to use relay (TURN) connectivity, even if STUN connectivity is available.
    },
}
```

- `authToken`
    - Type: `string`
    - Description: The participant auth token.
    - Required: Yes

- `defaults`
    - Type: `object`
    - Description: The default values for the session for the participant.
    - Required: No

- `onError`
    - Type: `function`
    - Description: This callback is called when an error occurs inside the SDK. Call to this function does not mean that the SDK is in a failed state. It is just a notification that an error occurred, which can be used to log the error or take other actions.
    - Required: No

- `overrides`
    - Type: `object`
    - Description: The overrides for the participant for the current session.
    - Required: No

:::note
It is recommended to not pass any options except `authToken` to the SDK, if you are not sure what each option does, or whether you need it.
:::
