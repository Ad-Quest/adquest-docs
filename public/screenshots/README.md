# Скриншоты для документации

Эта папка содержит скриншоты интерфейса AdQuest CRM для документации.

## Структура папок

```
screenshots/
├── crm/
│   ├── publisher/      # Скриншоты панели издателя
│   ├── advertiser/     # Скриншоты панели рекламодателя
│   ├── agency/         # Скриншоты панели агентства
│   └── auth/           # Скриншоты аутентификации
└── widget/             # Скриншоты виджета (будущее)
```

## Как заменить placeholder на реальные скриншоты

1. Сделайте скриншот нужной страницы в браузере
2. Оптимизируйте изображение:
   - Формат: PNG
   - Рекомендуемое разрешение: 1920x1080 или 2560x1440
   - Размер файла: < 500KB (используйте TinyPNG или аналог)
3. Замените соответствующий файл в нужной папке
4. Сохраните то же имя файла

## Рекомендации по созданию скриншотов

### Качество
- Используйте высокое разрешение экрана
- Убедитесь, что текст читаем
- Избегайте размытых изображений

### Содержание
- Используйте реалистичные данные (не тестовые "test123")
- Скройте чувствительную информацию
- Покажите типичный сценарий использования

### Оформление
- Убедитесь, что интерфейс в светлой теме (если не указано иное)
- Закройте лишние вкладки браузера
- Скройте панель разработчика

### Инструменты
- Встроенные инструменты ОС (Print Screen, Cmd+Shift+4)
- Расширения браузера (Awesome Screenshot, Nimbus)
- Специализированные программы (Snagit, Lightshot)

## Автоматическое обновление

После замены файлов:
1. Astro автоматически подхватит новые изображения
2. Перезагрузите страницу документации
3. Проверьте, что изображение отображается корректно
4. Проверьте функцию zoom (клик по изображению)

## Добавление новых скриншотов

Если нужно добавить новый скриншот:

1. Создайте файл в соответствующей папке
2. Добавьте компонент Screenshot в MDX файл:

```mdx
<Screenshot 
  src="/screenshots/crm/publisher/new-feature.png" 
  alt="Описание того, что показано на скриншоте"
  caption="Подпись под изображением (опционально)"
/>
```

3. Убедитесь, что путь начинается с `/screenshots/`

## Проблемы и решения

### Изображение не отображается
- Проверьте путь к файлу (должен начинаться с `/screenshots/`)
- Убедитесь, что файл находится в `public/screenshots/`
- Проверьте расширение файла (.png)

### Изображение слишком большое
- Сожмите через TinyPNG.com
- Уменьшите разрешение до 1920x1080
- Конвертируйте в WebP формат

### Zoom не работает
- Убедитесь, что плагин `starlight-image-zoom` установлен
- Проверьте, что изображение внутри компонента Screenshot
- Очистите кэш браузера
