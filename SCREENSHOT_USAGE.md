# Использование компонента Screenshot

## Описание

Компонент `Screenshot` предназначен для отображения скриншотов интерфейса в документации с автоматической поддержкой zoom функции.

## Возможности

- Автоматическое увеличение изображения по клику (через starlight-image-zoom)
- Опциональная подпись под изображением
- Адаптивный дизайн
- Lazy loading для оптимизации загрузки
- Поддержка темной и светлой темы

## Использование

### Базовый пример

```mdx
import { Screenshot } from "~/components";

<Screenshot 
  src="/screenshots/dashboard.png" 
  alt="Главная страница панели издателя" 
/>
```

### С подписью

```mdx
<Screenshot 
  src="/screenshots/add-site-form.png" 
  alt="Форма добавления нового сайта"
  caption="Форма добавления сайта с обязательными полями"
/>
```

### С указанием размеров

```mdx
<Screenshot 
  src="/screenshots/statistics.png" 
  alt="Страница статистики"
  caption="График доходов за последние 30 дней"
  width={1200}
  height={800}
/>
```

## Параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| src | string | Да | Путь к изображению |
| alt | string | Да | Альтернативный текст для accessibility |
| caption | string | Нет | Подпись под изображением |
| width | number | Нет | Ширина изображения в пикселях |
| height | number | Нет | Высота изображения в пикселях |

## Рекомендации

### Размещение скриншотов

Храните скриншоты в папке `public/screenshots/` с понятными названиями:

```
public/
  screenshots/
    crm/
      publisher/
        dashboard.png
        add-site-form.png
        statistics.png
      advertiser/
        campaigns.png
        budget.png
    widget/
      integration.png
      modes.png
```

### Оптимизация изображений

- Используйте PNG для скриншотов интерфейса
- Оптимизируйте размер файлов (рекомендуется < 500KB)
- Используйте разрешение 1920x1080 или 2560x1440 для Retina дисплеев
- Сжимайте изображения через TinyPNG или аналогичные сервисы

### Accessibility

- Всегда указывайте осмысленный `alt` текст
- Используйте `caption` для дополнительного контекста
- Описывайте что показано на скриншоте, а не "скриншот страницы"

Хорошо:
```mdx
<Screenshot 
  src="/screenshots/dashboard.png" 
  alt="Панель издателя с четырьмя карточками статистики: общий доход, активные сайты, показы сегодня и средний CTR"
/>
```

Плохо:
```mdx
<Screenshot 
  src="/screenshots/dashboard.png" 
  alt="Скриншот"
/>
```

### Аутентификация (`/crm/auth`)

- Форма входа

## Автоматический zoom

Компонент автоматически поддерживает увеличение изображения благодаря плагину `starlight-image-zoom`:

- Клик по изображению открывает его в полноэкранном режиме
- Повторный клик или ESC закрывает увеличенное изображение
- Поддержка жестов на мобильных устройствах
- Плавная анимация открытия/закрытия

Никаких дополнительных настроек не требуется.
